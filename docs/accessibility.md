# アクセシビリティ指針

AI Skill Map Generator では、「若手エンジニアがどの環境からでもストレスなく使えること」を目標に、以下のようなアクセシビリティ対応を行っています。

## 基本方針

- マウスだけでなく **キーボード操作でも主要な操作が行える UI** を目指す
- スクリーンリーダー利用者にも情報が伝わるように、意味のある要素には **適切なラベルと役割** を付与する
- 動きに敏感なユーザー向けに、**アニメーションを抑制できる仕組み** を用意する

## 実装している主な対応

- `components/ui/visually-hidden.tsx`
  - 画面上には表示せず、スクリーンリーダー専用のテキストを置くためのコンポーネントです。
  - アイコンボタンや視覚的なラベルに対して、読み上げ用の補足説明を付ける用途で使用しています。

- `components/ui/sr-announcer.tsx`
  - フォーム送信成功などの重要なイベント発生時に、スクリーンリーダーへ状態変化を通知するためのコンポーネントです。
  - ARIA ライブリージョンを利用して、「診断が完了しました」「結果を更新しました」といったメッセージを読み上げます。

- フォーム・ボタンラベル
  - 入力フォームやボタンには、意味の分かるテキストラベルを付与し、単なる「ここをクリック」ではなく **「スキルマップを生成する」** など具体的な動詞で表現しています。

- コントラストとサイズ
  - 主要なテキストとボタンは、WCAG が推奨するコントラスト比を意識して配色しています（薄いグレー上に薄い文字を重ねない）。
  - タップターゲットはおおむね **44px 以上** を目安に確保し、モバイルでもクリックしやすいサイズにしています。

## 今後の改善候補

- Lighthouse / axe-core などを使った自動アクセシビリティチェックの CI 連携
- 主要な画面に対するキーボードフォーカス順の調整と可視化
- `prefers-reduced-motion` に応じて、アニメーションをより積極的に無効化するオプションの追加
